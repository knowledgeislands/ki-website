# Ecosystem publication roadmap

## Blocking

Actively broken, or blocking the `Next` horizon: takes priority over everything else and must clear before `Next` work proceeds. Empty means nothing is on fire.

## Next

Scoped and ready to start — the immediate queue, picked up before anything in **Soon** or **Future**.

### Vendor canonical content from the three source repositories

The site publishes three source-owned bodies of material: philosophy and model from Arcadia Principal, practical tooling documentation from the harness, and normative portable contracts from KI Specifications. Establish explicit, source-labelled vendor inputs for all three, so the site remains independently deployable without becoming a parallel authority. The first Arcadia body of content is the KB tool-ecosystem documentation from the `Streams/Active/Agentic Tool Documentation Proposal` stream.

## Soon

Understood and roughly scoped but not yet started — worth doing once the **Next** queue clears, ahead of anything still speculative.

## Waiting for

Worth doing, but presently blocked on an external dependency or decision. Revisit when its named condition changes rather than treating it as dormant local work.

## Future

Speculative or not yet scoped — items marked _(candidate)_ need a scoping pass (or a decision to drop them) before they're actionable.
